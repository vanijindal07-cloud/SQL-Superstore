# SQL-Superstore
# SQL Basics – Task 3 (Superstore Analysis)

## Tool Used
- DB Fiddle (MySQL 5.7)

## Dataset
- Superstore dataset (manually created and inserted)
- Includes categories like Electronics, Kitchen, Furniture, Clothing, Storage, Decor
- Columns: order_id, order_date, customer_name, region, category, sub_category, sales, profit

## Objective
To practice SQL basics including filtering, sorting, aggregation, grouping, and exporting query results.

## SQL Tasks Performed
1. Created Superstore table with appropriate data types
2. Inserted multiple records to simulate real sales data
3. Used SELECT queries to explore dataset
4. Applied WHERE clause for filtering data
5. Used ORDER BY to sort sales values
6. Applied GROUP BY with SUM and AVG for aggregation
7. Used HAVING clause to filter aggregated results
8. Used BETWEEN for date-based filtering
9. Used LIKE for pattern matching
10. Exported SQL query results into CSV format

- queries_task3.sql → Contains all SQL queries
  
-sales_summary.csv → Exported query results
